# CodeBook Lab documentation

Source for the CodeBook Lab package docs site, published at
<https://lorcanmclaren.com/codebook-lab>. It is a standalone Quarto **website**
project that renders into `../docs/codebook-lab/` and is excluded from the parent
academic-site build.

## Structure

- Hand-written prose pages: `index.qmd`, `install.qmd`, `tasks.qmd`,
  `annotation-types.qmd`, `conditionals-retries.qmd`, `experiments.qmd`,
  `outputs-metrics.qmd`, `human-reliability.qmd`, `citation.qmd`.
- `reference/` — the API reference, **auto-generated by
  [quartodoc](https://machow.github.io/quartodoc) from the `codebook_lab`
  docstrings**. Do not edit these files by hand; edit the docstrings instead.
- `styles.css` — fonts/colours layered over the shared `simplex` +
  `../custom-{light,dark}.scss` theme.

## Building

`quartodoc build` regenerates `reference/` from the package docstrings and must
run before quarto. Use the `Makefile` so it always does:

```bash
pip install -r requirements.txt   # quartodoc + the package
make render                       # quartodoc build, then quarto render
make preview                      # quartodoc build, then quarto preview
```

`quartodoc build` is **not** a Quarto `pre-render` hook on purpose: it rewrites
`reference/` and the watched `_sidebar.yml` on every pass, which sends
`quarto preview` into an endless re-render loop. Keeping it a separate step (run
once up front) avoids that while content edits still live-reload. After editing a
**docstring**, rerun `make reference` to refresh the API pages.
